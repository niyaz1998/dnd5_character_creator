# Code Style

Мы придерживаемся рекомендаций, описанных в [Effective Dart](https://dart.dev/guides/language/effective-dart/design)

Перед каждым пулл реквестом необходимо отформатировать код в измененных файлах с помощью dartfmt.(flutter format)

# Именование

## Файлы

Используется в snake_case.
Название файла должно иметь следующую структуру.

[раздел]_[тип].dart

Пример: user_details_screen.dart, shop_entity.dart

## Классы

Название классов UpperCamelCase.
Для создание приватыных классов используем префикс _

Название класса в конце должно содержать в себе тип.

Пример: UserEntity, AdultDialog

## Переменные и методы

Название переменных и методов lowerCamelCase.

### Методы
Название метода в начале должно содержать в себе действие(глагол)
get,put,update,delete итд

Название метода не должно содержать в себе And/Or
и метод соответственно не должен выполнять подобную логику.

### Переменные

Константы именнуются также lowerCamelCase.
